通过less编写自己的栅格系统

发表于 2015-12-28   |   分类于 css   |  

栅格系统

用过bootstrap的都知道他的栅格系统，可以非常简便地给页面布局带来许多灵活性。我们只用在html文件中为元素指定特定的栅格类就可以精确地在水平方向上布局元素，再加上自己指定一些css规则，就完成了页面的布局。但是用久了发现他提供的12个栅格有时候并不能满足项目需求，想要更灵活精确一些，就需要对他动些手脚了（当然并不只bootstrap有栅格系统，很多css框架比如说pure都有栅格系统）。最简单也是最不应该用的方法是直接改他的css源文件，或者覆盖，那么我想说的是在bootstrap3.3.5的css文件中，跟grid系统相关的源代码从1585行到2251行，没错，刚好是666行，不信你自己去看。

应该没人愿意去改600多行代码来扩展他吧，就算你改了后期也根本没法维护。因此必须要想其他的办法。有个方法就是用其他框架，比如说pure的栅格系统有24列，但是毕竟那不是你自己的东西，而且不能就为了用pure的栅格系统在项目里又用bootstrap又用pure吧。所以为啥不自己写一个想有多少列就有多少列的栅格系统呢。那么问题就来了，要怎么写？这时候就可以看看人家是怎么写的了。

less

bootstrap的css是用less写的，less是一种css预处理语言，是动态的。他解决了css的最大短板，不能像编程语言一样使用变量函数等。写css给人感觉是在写口水话，没有在编程，而less解决了这个问题。同样的比较出名的css预处理语言还有sass。这种大型库没有预处理语言是不可能维护的。有了less，在编写好less文件之后，只用将less文件编译成css文件即可使用（给人感觉有点像es6编译成es5）。less的语法其实就是在css上的扩展，支持原生的css写法。他的几个核心语法并不难，但是整体挺杂的，感觉不是所有语法都常用，这是官方文档。

实现

要编写自己的栅格系统，可以直接改写bootstrap提供的less源文件。但是bootstrap这个库比较大，关于grid的就有一个主文件和两个mixin文件。做为一个新手实在是hold不住也不能完全理解，所以干脆就自己写一个好了。

基本思路

既然看他的less源码不好理解，那么就直接看css文件，然后再自己写出能输出那样css文件的less源码。虽然有666行，但是很多都是重复的代码（再次从编程的角度说明css有多辣鸡），而且非常好理解，所以可以从这里入手。其实要用到less语法的地方只有以下几个地方：

  1. 为了之后好维护，需要用到变量，设定一些全局变量用到grid system里（列数、媒体查询中的设备宽度等）
  2. 由于css中会有巨量的重复，所以会用到less的循环
  3. less中的循环是通过mixin加上guards然后传入不同参数调用自己实现的，因此还需要mixin

因此就光编写grid system来说，用到的less语法很少，但是带来的好处是非常巨大的。因为有变量存在，因此在之后想添加或者减少栅格系统的列数就只用改一个地方，节约大量时间。如果想改栅格类的名字也很简单，只用在mixin中改一个地方，css中上百行的代码都被改过来了。

编译

用less等预处理语言需要用编译器将其编译成css之后才能用，虽然官方提供了在客户端编译的方法，但是能够预先编译的话何必浪费客户端宝贵的资源。所以用node来编译less是最常用的方法，官方有大量插件供使用。同样node下的自动化构建工具例如gulp和grunt也有成熟的less插件，可以配合其他插件使用，一步到位更加酸爽。

代码

说了这么多可以看代码了，首先是存放全局变量的文件：

// variables.less

@numOfColumns: 24;            // 栅格系统的列数
@smWidth: 768px;                // 小屏幕设备
@mdWidth: 992px;                // 中屏幕设备
@lgWidth: 1200px;                // 大屏幕设备

然后是我们的栅格系统主文件：

// grid.less

@import "variables.less";    // 导入全局变量

.km-container {                        // container的css表达
  padding-right: 15px;
  padding-left: 15px;
  margin-right: auto;
  margin-left: auto;
}
.km-row:before,                        // 来自bootstrap的css文件中对row的描述
.km-row:after {
  display: table;
  content: " ";
}
.km-row:after {
    clear: both;
}
.km-row {
  margin-right: -15px;
  margin-left: -15px;
}

// 调用mixin输出css
.initColumns(@numOfColumns);

.makeColumns(@numOfColumns, xs);        // 输出km-xs-1 ~ km-xs-@{numOfColumns}的css表达
@media (min-width: @smWidth) {
    .makeColumns(@numOfColumns, s);        // 输出km-s-1 ~ km-s-@{numOfColumns}的css表达
}
@media (min-width: @mdWidth) {
    .makeColumns(@numOfColumns, m);        // 输出km-m-1 ~ km-m-@{numOfColumns}的css表达
}
@media (min-width: @lgWidth) {
    .makeColumns(@numOfColumns, l);        // 输出km-l-1 ~ km-l-@{numOfColumns}的css表达
}


// mixins定义

// 设置所有列的共有规则
.initColumns(@n) {
    .wrapSelectors(@i: 1) {
        @selecs: ~".km-xs-@{i}, .km-s-@{i}, .km-m-@{i}, .km-l-@{i}";
        .wrapSelectors(@selecs, @i + 1);
    }
    .wrapSelectors(@item, @i) when (@i =< @n) {
        @selecs: ~".km-xs-@{i}, .km-s-@{i}, .km-m-@{i}, .km-l-@{i}";
        .wrapSelectors(~"@{item}, @{selecs}", @i + 1);
    }
    .wrapSelectors(@item, @i) when (@i > @n) {
        @{item} {
        position: relative;
          min-height: 1px;
          padding-right: 15px;
          padding-left: 15px;
        }
    }
  .wrapSelectors();
}


.makeColumns(@n ,@class) {

    // 设置所有相同class的类的共同规则
    .setAll(@i: 1) {
        @selecs: ~".km-@{class}-@{i}";
        .setAll(@selecs, @i + 1);
    }
    .setAll(@item, @i) when (@i =< @n) {
        @selecs: ~".km-@{class}-@{i}";
        .setAll(~"@{item}, @{selecs}", @i + 1);
    }
    .setAll(@item, @i) when (@i > @n) {
        @{item} {
            float: left;
        }
    }
    .setAll();

    // 设置对应class中每个不同大小列的宽度
    .setEachWidth(@i: 1) when (@i =< @n) {
        .km-@{class}-@{i} {
            width: (@i * 100% / @n);
      }
        .setEachWidth(@i + 1);
    }
    .setEachWidth();

    // 设置对应class中每个不同大小的offset的左margin（没有实现bootstrap中的pull和push类，因为个人不太爱用）
    .setEachOffset(@i: 0) when (@i =< @n) {
        .km-@{class}-offset-@{i} {
            margin-left: (@i * 100% / @n);
      }
        .setEachOffset(@i + 1);
    }
    .setEachOffset();
}

可以看出来，通过100行的less代码可以轻易产出600多行的css代码，并且之用修改全局变量的值，就可以动态编译成css代码。

总结

虽然刚开始接触less，但是可以看出来类似的css预处理语言会给我们带来巨大的效率提升。虽然学习此类语言会花一点时间，但是他们节约的时间绝对会比学习他们需要的时间多得多。有了这种预处理语言，可以为项目的构建、维护和修改带来极大的便利。

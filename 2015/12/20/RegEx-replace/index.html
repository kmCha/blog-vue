String类的replace()方法妙用：正则表达式+函数

发表于 2015-12-20   |   分类于 javascript   |  

在原生javascript中，有一个基本每个开发者都知道甚至经常使用的字符串方法：replace()。他通常被用于一个很简单的情况：将字符串中的某些字符替换为指定字符。正因为他的用法很简单，所以很容易被低估。

普通用法

通常，我们（新手）用replace()的时候，大多数时候会传入调用该方法的字符串的一个substring作为该方法的第一个参数，也就是直接将第一个参数指定的子字符串替换为你指定的第二个字符串。这样做对于非常非常（I mean，真的非常）简单的字符串操作是没有问题的，但是字符串一复杂，比如你要替换字符串中多个位置的字符就显得有点力不从心了。但是因为replace()的第一个参数支持正则表达式，所以有了第二种文艺用法。

文艺用法

将第一个参数换成正则表达式之后，对字符串的操作就灵活了许多。我们可以指定匹配正则表达式来选取需要被替换的字符，但是光用正则表达式作为第一个参数实际上意义也不是很大，因为通常我们替换的时候需要知道正则表达式中抓取的某些片段是什么，才能决定第二个参数传入什么字符串来代替。replace()为我们提供了$操作符，我们能使用$1,$2等来获取第一个捕获的引用，第二个捕获的引用……因此，我们可以在全局正则每次捕获之后获取捕获的字符串的引用，从而放入第二个参数中。什么意思？先来个例子：

var style = "borderTopRightRadius";
style.replace(/([A-Z])/g, "-$1").toLowerCase() === "border-top-right-radius"; //true

上面只用了两行代码就将一个驼峰式表示的字符串转换成了用横线分隔的字符串。试想一下用普通方法来实现这个，还需要遍历整个字符串来替换符合要求的字符。在这里，我们用/([A-Z])/g表达式来匹配style中的大写字母（这里是T，R，R），因为他们被放在了捕获分组中，因此可以用$1来读取每次捕获的引用，并且加上-符号放入第二个参数中，即：第一次将T替换成了-T，第二次将R替换成了-R，第三次将R替换成了-R。然后用返回的新字符串调用toLowerCase()方法，将字符串转换为全部小写，最后就得到了border-top-right-radius字符串。但是，replace()方法的第二个参数其实是支持函数的。因此，对字符串的操作还能够更灵活。

高逼格用法

当replace()方法的第一个参数是正则表达式，第二个参数是函数的时候，对正则表达式中的每一个捕获，都会调用一次函数，并且会有若干个参数传入函数的参数列表

  1. 匹配的完整文本
  2. 匹配的捕获，一个捕获对应一个参数
  3. 匹配字符串在源字符串中的索引
  4. 源字符串

函数需要有个返回值作为replace()的第二个参数。
有了这个特性，我们当然可以用函数来实现文艺用法中的例子，只用将第二个参数换成能够处理传入捕获引用的函数就行了。在这里不写出来了。下面来说一个高逼格用法能够实现的高逼格查询字符串压缩方案。假如有一个查询字符串长这样：foo=1&foo=2&blah=a&blah=b&foo=3，我们想将它压缩成这样foo=1,2,3&blah=a,b，看如下例子：

//代码来源：John Resig & Bear Bibeult, javascript ninja, 7.10
function compress(source) {
    var keys = {};                //保存每个捕获的键

    source.replace(                //利用replace遍历字符串
        /([^=&]+)=([^&]*)/g,
        function(full, key, value) {
            keys[key] = (keys[key] ? keys[key] + "," : "") + value;  //将相同key的值放到keys对象的同一个key中形成字符串
            reture "";                //返回的字符串无关紧要
        }
    );

    var result = [];            //存放最终压缩结果的数组
    for (var key in keys) {
        result.push(key + "=" + keys[key]);
    }

    return result.join("&");        //将结果数组转换成用&分隔的字符串
}

compress("foo=1&foo=2&blah=a&blah=b&foo=3") === "foo=1,2,3&blah=a,b";    // ture

以上例子利用replace()来遍历字符串而不是用它来得到最终结果，因此最后的返回值设什么都无所谓，我们只是利用正则表达式每次捕获都会调用函数这一特性来遍历整个字符串来对我们想操作的子字符串进行我们想要的操作。例如这个例子中，我们匹配原字符串中的=符号以及他前后的字符串，key为=前面的字符串，value为=后面的字符串。我们在函数中将每次捕获的相同key值的结果融合到了keys对象中对应的键中，最后遍历keys对象将他的键值对保存在result数组中，最后再将数组转换为字符串返回。
由此可以看出，一个简简单单的replace()方法可以用正则表达式和函数为所欲为，为字符串的操作带来了极大的灵活性（和高逼格）。在页面脚本的编写中可以优雅的使用，毕竟现在写个程序都在比谁更优(zhuang)雅(bi)。

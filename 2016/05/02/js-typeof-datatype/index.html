js数据类型和typeof语句

发表于 2016-05-02   |   分类于 javascript   |

JavaScript数据类型

众所周知，JavaScript的几种数据类型分别是number, string, null, undefined, boolean, object和symbol(ES6新增的)。这里我们就说前面6种，symbol先不说（其实是因为我还没仔细研究过 ˇˍˇ )。
在前面6种数据类型中，前5种为基本类型，object为引用类型。object中又包含了一些其他的数据类型，比如函数、Date实例、正则等等。

typeof语句

JavaScript中，免不了有判断数据类型的时候，这时候我们一般会想到typeof语句。下面是他的基本用法：

typeof 1; //“number”
typeof “1”; //“string”
typeof null; //“object”
typeof undefined; //“undefined”
typeof true; //“boolean”
typeof {“1”:1}; //“object”

上面的例子对应之前说的6种数据类型的判断，但是问题来了，其他的还好，但是那个null返回的不是”null”而是”object”，跟我们期望的结果不太一样。
这个bug打JavaScript出生的时候就存在了，而之所以这么久没有修正的原因，是因为如果修正了这样的bug，那已经依赖这个bug来开发的程序会受到影响，所以修复之后会带来更多的问题，因此就这样将错就错了。其根本的原因是JavaScript在底层将二进制表示以0开头的数据类型判断为object，而null的二进制表示全部为0，因此就被判定成了object。

object中的另类：function

上面我们说到，object中又包含了许多其他的数据类型，函数、正则、Date实例等等，所以当我们对这些被包含的数据类型用typeof的时候，按照我们的期望应该是返回”object”， 看下面

typeof new Date(); // “object”
typeof /^&/; // “object”
typeof function a(){}; // “function”

所以我说function是个奇葩，明明就属于object，但是用typeof语句的时候返回的又是自己独有的function而不是object。

JavaScript未定义(undefined)和未申明(undeclared)

在JavaScript中，有一种数据类型叫做undefined，字面意思是“未定义”。这个“未定义”很容易和我们的“未申明”搞混。实际上他们是不同的两个东西。

未定义和未申明

当一个数据类型是undefined的时候，通常是我们已经申明了一个变量，但是没给变量赋值的情况。而如果一个变量没有被申明，引用这个变量的时候就会抛出referenceError，从而程序停止运行。

因为有以上的区别，我们在程序中判断一个变量有没有值的时候就不太安全，通常情况下，我们期望判断一个变量是不是undefined，就像这样:

if(a !== undefined) {
// do something
}

如果我们申明了a变量并且没有给它赋值的话，上面的if语句成立，执行下面块中的语句。但是如果我们的a没有申明的话，程序就会抛出错误，停止执行。因此这个方法不太安全。

typeof可以解决上述问题

同样是上面的例子，如果我们将程序改一下：

if(typeof a !== “undefined”) {
// do something
}

程序就安全了。因为typeof语句后面的变量如果没有被申明的话，也会返回“undefined”字符串。这样就可以判断一个变量是否为undefined或者根本没有被申明而不会抛出错误打断程序的运行了。

因此，typeof表达式是有一些小坑，输出的值可能有时候跟我们期望的不太一样，但是它也是有可取之处的。

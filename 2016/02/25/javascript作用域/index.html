javascript作用域

发表于 2016-02-25   |   分类于 javascript   |  

距离上次写博客又有一段时间了，今天趁着assignment写的差不多了又不想搞project的空闲来总结下对JavaScript中的作用域的理解。

作用域是什么东西

作用域在我们的代码中处处可见，许多重要的概念也与作用域息息相关，比如说闭包。而说到作用域，每个人都知道他是个什么，有什么用，但是真正要描述的时候，说不清。那么作用域到底specifically是个啥呢？要说明这个问题，就要把JavaScript引擎和编译器一起拖出来讲。

JavaScript引擎、编译器和作用域的关系

说到JavaScript，出现最多的词可能是“动态类型语言”，人们往往忘了它也是一门“编译型语言”。当然人们不经常提起“编译型语言”也有他自己的原因，他跟一般的编译型语言不太一样。JavaScript的编译发生在代码运行前的一瞬间，也就是说编译之后马上就运行了。虽然编译和运行几乎同时进行，但是实际上还是存在先编译后运行这一事实的。那么要说明JavaScript引擎、编译器和作用域的关系，我们先来看一段代码：

var a = 1;

这段代码是一条最简单的变量申明赋值语句，虽然简单，但是联系到了JavaScript引擎、编译器和作用域三个大佬。
上面的语句分成了两个步骤，第一个步骤是var a也就是申明a变量，第二个步骤是a = 1也就是将1赋给刚刚申明的a变量。其中，变量申明发生在代码编译阶段，代码赋值发生在代码运行阶段。这句话就是下面这段代码输出undefined而不是抛出ReferenceError的原因：

console.log(a); // undefined
var a = 1;

翻译成代码就是：

var a;
console.log(a); // undefined
a = 1;

知道了变量申请和赋值的机制，下面来看看JavaScript引擎、编译器和作用域是怎么被联系到一块的。
首先是变量申请阶段，这时候编译器会询问作用域中是否有a变量存在，如果存在，则忽略这条申请，如果不存在，则在作用域中创建一个a变量。然后编译过程执行完毕以后，进入代码执行阶段。在代码执行阶段中，JavaScript引擎看到a=1赋值语句后，会去查找作用域中是否存在一个a变量，如果存在，则把它的值变更为1，如果不存在，则在全局作用域中创建一个a变量并且赋值为1。
因此，可以说作用域是决定怎么和在哪里访问变量的一个机制，它是JavaScript引擎和编译器间的桥梁。

作用域的种类

通常作用域有词法作用域和动态作用域两类。然而，在JavaScript中，只存在词法作用域。

动态作用域

动态作用域是指，作用域是代码执行阶段建立的作用域，在写代码的时候是不能确定的。虽然JavaScript中没有动态作用域，但是this的行为跟动态作用域就很像，this的值是根据代码运行时函数被调用的位置来决定的，而不是根据写代码时的代码结构决定的。

词法作用域

词法作用域是指，在代码编译的词法分析阶段创建的作用域，是根据写代码时候的代码结构来决定的。跟其他很多编程语言一样，JavaScript中采用的作用域就是词法作用域。词法作用域是闭包这一现象的根本，关于闭包见我之前的文章–从原理来看javascript闭包－作用域链。

块作用域

在JavaScript中，是不存在块作用域的，准确点来说是ES6到来前是不存在的，再准确点就是在ES6到来前除了利用try catch语句等反人类方法来模拟块作用域之外，是不存在的。

因为JavaScript中，作用域是函数来创建的。除了全局作用域之外，其他作用域基本都是各个函数作用域。而函数作用域一层一层嵌套，就成了作用域链。明明再说块作用域的咋跑作用域链去了。。。没事就提一下，回到块作用域。
因为没有块作用域这个实用的东西，那么就相当于挖了许多坑来让新手跳。例如从原理来看javascript闭包－作用域链中讲的那个新手坑，就是没有块作用域带来的副作用。然而问题并不大，只要掌握了作用域的原理和闭包，这样的坑来一个能填一个。

ES6中出现了let和const关键字能够创建块作用域，能够帮助喜欢使用块作用域的用户更加顺手地写代码，对那些人算是一定程度上的官方填坑，喜不喜欢还得看习惯，个人认为还是挺实用的。

思路有点混乱，写得应该也很混乱，先到这里以后想起了再添。

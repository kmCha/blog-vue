javascript设计模式之状态模式(state)

发表于 2016-01-17   |   分类于 javascript   |  

状态模式是什么？

现实生活中，很多事物都有状态这一说，比如说开关，有开和关这两个状态，当然高档一些的开关可能有很多个档，每个档都对应一个不同的状态。而在不同的状态下按开关，他的行为是不一样的。这就是状态模式的核心。状态模式把一个有多个状态的对象定义为上下文(context)对象，把他里边的各种状态都封装成状态对象。在用状态模式之前，我们可能觉得表示一个对象的状态还不简单？给他个state属性，给state属性赋不同的字符串值来表示不同的状态不就行了，要想要一个动作在不同的状态下有不同的效果，通过判断state的值再执行不同的动作就行了。是的没错，但这也是为什么要引入状态模式的原因。用前边说的用对象的字符串属性表示状态再来根据字符串确定要执行的动作是可以达到跟用状态模式的代码一样的效果，但是对象中会存在大量的if判断语句，当对象的状态变的很多的时候，维护和可读性就是一个大问题。

一个例子

下面举个例子，如果我们要编写一个文件上传模块，那么上传的过程中，我们有个暂停/继续的按钮，那么这个按钮根据文件上传过程中的状态不同会表现出不同的功能。比如文件扫描完成后，需要点击开始上传，那么这个按钮就会表现出开始上传的功能。当文件开始上传之后，那么这个按键应该显示出“暂停”字样，并且表现出暂停上传文件的功能。我们还可以添加第二个删除文件的按钮，在文件还没开始上传或者在上传的过程中时，这个按钮是禁用的，当暂停或者上传成功之后，激活这个按钮。

不用状态模式的情况

下面就先来看看用state属性和其字符串值来表示对象的状态是什么情况：

var Upload = function() {
    this.state = "initial";
    this.button1 = null;
    this.button2 = null;
};
Upload.prototype.init = function() {
    var button1 = document.createElement("button"),
        button2 = document.createElement("button");

    button1.innerHTML = "开始上传";
    button2.innerHTML = "删除文件";
    button2.disabled = true;
    document.body.appendChild(button1);
    document.body.appendChild(button2);

    this.button1 = button1;
    this.button2 = button2;

    this.bindEvent();
};
Upload.prototype.bindEvent = function() {
    var _self = this;

    this.button1.onclick = function() {
        if(_self.state === "initial") {
            _self.changeState("uploading");
        }
        else if(_self.state === "uploading") {
            _self.changeState("pausing");
        }
        else if(_self.state === "pausing") {
            _self.changeState("uploading");
        }
        else if(_self.state === "finished") {
            console.log("上传完成");
        }
    };
    this.button2.onclick = function() {
        if(_self.state === "initial") {
            console.log("还没上传不能删除");
        }
        else if(_self.state === "uploading") {
            console.log("正在上传不能删除");
        }
        else if(_self.state === "pausing") {
            _self.changeState("initial");
        }
        else if(_self.state === "finished") {
            _self.changeState("initial");
        }
    };
};
Upload.prototype.changeState = function(state) {
    switch(state) {
        case 'initial':
            this.state = "initial";
            this.button1.innerHTML = "开始上传";
            this.button2.disabled = true;
            break;
        case 'uploading':
            this.state = "uploading";
            this.button1.innerHTML = "暂停";
            this.button2.disabled = true;
            console.log("文件开始上传");
            break;
        case 'pausing':
            this.state = "pausing";
            this.button1.innerHTML = "继续";
            this.button2.disabled = false;
            console.log("文件暂停上传");
            break;
        case 'finished':
            this.state = "finished";
            this.button1.innerHTML = "开始上传";
            this.button2.disabled = false;
            console.log("文件上传完毕");
            break;
    }
};

通过这种方式，我们在bindEvent()方法中通过各种if语句判断了在按钮点击的时候upload实例对象此时的state属性的值，然后根据他的值来确定按钮点击后应该进入的状态，并且调用upload实例对象的changeState()方法并且传入下一个状态。在changeState()函数中，需要根据传入的状态进行相应的操作。
这样的代码在之后的维护过程中是很痛苦的，比如我们之后要加一个状态给upload对象，那么我们既要修改button1和button2的onclick()函数中的条件语句，又要在changeState()函数中加一个case语句，这在已经有很多条件语句了的情况下是很难进行改动的。

用状态模式的情况

在javascript中实现状态模式，我们可以模拟传统面向对象语言中的实现方法，即定义一个抽象的状态类，然后定义不同的状态类来继承这个抽象的状态类来重写其中的接口，但是在javascript中，对象可以不通过类来产生，因此我们可以直接写一个有限状态机(finite state machine)对象，将我们的不同状态对象包装在里面，用上下文对象的属性来引用不同的状态对象来实现不同的状态。
下面看代码：

var Upload = function() {
    this.currState = fsm.initial;
    this.button1 = null;
    this.button2 = null;
};
Upload.prototype.init = function() {
    var button1 = document.createElement("button"),
        button2 = document.createElement("button");

    button1.innerHTML = "开始上传";
    button2.innerHTML = "删除文件";
    button2.disabled = true;
    document.body.appendChild(button1);
    document.body.appendChild(button2);

    this.button1 = button1;
    this.button2 = button2;

    this.bindEvent();
};
Upload.prototype.bindEvent = function() {
    var _self = this;
    this.button1.onclick = function() {
        _self.currState.process1.call(_self, arguments);
    };
    this.button2.onclick = function() {
        _self.currState.process2.call(_self, arguments);
    };
};
Upload.prototype.upload = function() {
    console.log("文件开始上传");
    this.button1.innerHTML = "暂停";
    this.button2.disabled = true;
    this.currState = fsm.uploading;
};
Upload.prototype.pause = function() {
    console.log("文件暂停上传");
    this.button1.innerHTML = "继续";
    this.button2.disabled = false;
    this.currState = fsm.pausing;
};
Upload.prototype.remove = function() {
    console.log("文件已删除");
    this.button1.innerHTML = "开始上传";
    this.button2.disabled = true;
    this.currState = fsm.initial;
};
Upload.prototype.finish = function() {
    console.log("文件上传完成");
    this.button1.innerHTML = "开始上传";
    this.button2.disabled = false;
    this.currState = fsm.finished;
};

var fsm = {
    initial: {
        process1: function() {
            this.upload();
        },
        process2: function() {
            console.log("还没上传文件不能删除");
        }
    },
    uploading: {
        process1: function() {
            this.pause();
        },
        process2: function() {
            console.log("文件上传过程中不能删除");
        }
    },
    pausing: {
        process1: function() {
            this.upload();
        },
        process2: function() {
            this.remove();
        }
    },
    finished: {
        process1: function() {
            console.log("上传完成");
        },
        process2: function() {
            this.remove();
        }
    }
};

现在我们把状态都包装在了fsm状态机对象中，直接从upload对象的currState属性引用状态对象，并且在bindEvent()方法里将currState属性引用的状态对象的接口绑定成按钮的onclick回调函数。在不同的状态对象中，接口的名字是相同的，这样才能保证绑定事件处理函数的一致性，另外，不同的状态对象中的方法会调用不同的定义在upload对象中的各种方法，来达到不同的操作目的。
这样的代码中没有一个if语句，所以很容易看懂，之后也好维护，比如想要加一个状态，直接在fsm状态机对象中添加一个状态对象，再按照情况给upload对象加一个给状态对象调用的方法用来转换状态即可。

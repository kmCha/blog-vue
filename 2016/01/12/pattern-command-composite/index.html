javascript设计模式之命令模式(command)和组合模式(composite)

发表于 2016-01-12   |   分类于 javascript   |  

什么是命令模式？

有些时候，需要向某些对象发送请求，但是并不知道接收请求的对象是谁，或者不知道接收者应该怎么对请求作出反应，这时候，为了解除发送请求和接收请求双方的耦合，就可以用命令模式。咋听起来这么像观察者模式？我也觉得

好了，既然命令模式和观察者模式会以两个不同的名字同时存在，那他们的使用方式或者原理肯定是不同的。在观察者模式中，消息的发送方可以在合适的时候将消息发送至中间的中介对象，然后中介对象再根据消息来调用订阅了该消息的函数。而在命令模式中，命令是在发送命令的对象中执行的，然后命令再传送到命令的接收者(receiver)对象上。他们都通过某种方式实现了两个对象间的解耦。

命令模式的核心——命令对象

命令(command)对象，通常是一个具有receiver属性和exec()方法的对象。在receiver属性中装的是命令的接收对象。exec()方法就是暴露给命令的执行对象的方法。命令的执行者只用调用命令对象的exec()方法，至于怎么将命令传达给命令的接收方，就是命令对象的事了。对于不同的操作，都会有与之对应的命令对象，这句话的意思是虽然命令的执行对象不知道命令的接收对象，但是命令对象必须要知道命令的接收对象的相应方法的调用方式。比如说有两个按钮，一个负责执行图片的加载，另一个负责菜单的刷新，那么用命令模式写出来就是：

<body>
    <button id="load">加载图片</button>
    <button id="refresh">刷新菜单</button>
</body>

<script type="text/javascript">
    var LoadCommand = function(receiver) {
        this.receiver = receiver;
    };
    LoadCommand.prototype.exec = function() {
        this.receiver.load();
    };

    var RefreshCommand = function(receiver) {
        this.receiver = receiver;
    };
    RefreshCommand.prototype.exec = function() {
        this.receiver.refresh();
    };

    var loadImg = {
        load: function() {
            // 加载图片
        }
    };
    var refreshMenu = {
        refresh: function() {
            // 刷新菜单
        }
    };

    var loadCommand = new LoadCommand(loadImg);
    var refreshCommand = new RefreshCommand(refreshMenu);

    var setCommand = function(id, command) {
        document.getElementById(id).onclick = function() {
            command.exec();
        };
    };

    window.onload = function() {
        setCommand("load", loadCommand);
        setCommand("refresh", refreshCommand);
    };
</script>

那么接下来点击加载图片按钮就会执行loadCommand.exec()，点击刷新菜单按钮就会执行refreshMenu.exec()。因为每个命令对象都有exec()方法，所以我们可以用一个通用的setCommand()函数将命令对象绑定给执行命令的操作。

命令模式的好处

上面那个例子显然说明不了命令模式的优势，将操作封装在一个命令对象中，然后再传入setCommand()函数，what’s the point? 确实，当时看得我也是一头雾水，与其写这么麻烦，我这么写不一样的吗：

<script type="text/javascript">
    var loadImg = function() {
        // 加载图片
    };
    var refreshMenu = function() {
        // 刷新菜单
    };

    var setCommand = function(id, fn) {
        document.getElementById(id).onclick = function() {
            fn();
        };
    };
    window.onload = function() {
        setCommand("load", loadImg);
        setCommand("refresh", refreshMenu);
    };
</script>

确实，这样写也能实现相同的效果，因为在javascript里头函数是一型对象，需要在传统面向对象语言中传来传去的命令对象在javascript里可以直接传函数，这是语言自带的优势，因此在javascript中命令模式是一种隐形的模式。
但是，将命令的接收者包装成命令对象确实是有好处的，比如说命令模式的高级用法：exec()的逆操作undo()和运用队列缓存命令确保一个命令执行完毕以后才执行下一个等等。

智能命令

第一个例子中，我们将命令的接收者包装在了命令对象中，由命令对象来负责调用他，命令对象暴露一个exec()方法来让命令的执行者执行命令。第二个例子中，我们直接将第一个例子中的命令的接收者直接作为命令本身传入了命令的执行者之中，命令执行者直接调用命令的接收者来执行命令，这是javascript语言带来的便利。但是我们说了对于命令本身而言，暴露出相同的接口（比如exec()和undo()等）是很重要的，因此我们需要将以上两个例子结合一下，就形成了我们的智能命令：

<script type="text/javascript">
    var loadImgCommand = {
        exec: function() {
            // 加载图片
        }
    };
    var refreshMenuCommand = {
        exec: function() {
            // 刷新菜单
        }
    };

    var setCommand = function(id, command) {
        document.getElementById(id).onclick = function() {
            command.exec();
        };
    };
    window.onload = function() {
        setCommand("load", loadImgCommand);
        setCommand("refresh", refreshMenuCommand);
    };
</script>

可以看到，本例中，有第一个例子的命令对象，但是区别在于这里的命令对象里面并没有receiver属性，取而代之的是exec()方法中直接执行命令接受者需要做的事情，也就是我们第二个例子中传入命令执行者的函数。这样的命令对象会执行命令接受者做的事情，而不是像智障一样传给命令的接收者做，因此叫他智能命令。

什么是组合模式

组合模式就是说一个对象是由其他对象所组成的，适用于对象间的关系类似于树的情况。树中的节点分为两种，叶子节点和组合节点，非叶子节点的就是组合节点。在组合模式中，节点类型是对外透明的，意思就是所有节点对外暴露的接口都是相同的，命令模式的exec()在这里就有用武之地了。因此用户是不知道节点到底是叶子节点还是组合节点的，保持这样的透明性有利于封闭——开放原则。

宏命令 ＝ 命令模式 ＋ 组合模式

宏命令就是许多命令的集合体，比如说魔兽世界在插件中设置了宏之后，原本的12345无限循环变成只用逮着1啪啪啪按不停就行了。在这里，想要执行一连串的命令，那么就可以结合命令模式和组合模式，将需要执行的命令作为根节点的叶子节点形成一棵树，然后只用执行根节点的exec()方法就可以遍历整个树挨个执行叶子节点了。有点像深度优先搜索。下面还是用之前的例子，看用宏命令该怎么写：

<body>
    <button id="both">组合命令</button>
</body>

<script type="text/javascript">
    var MacroCommand = function() {
        var commands = [];
        return {
            add: function(command) {
                commands.push(command);
            },
            exec: function() {
                for(i = 0, l = commands.length; i < l; i++) {
                    commands[i].exec();
                }
            }
        };
    }

    var loadImgCommand = {
        add: function() {
            throw new Error("叶子节点不能添加子节点！");
        },
        exec: function() {
            // 加载图片
        }
    };
    var refreshMenuCommand = {
        add: function() {
            throw new Error("叶子节点不能添加子节点！");
        },
        exec: function() {
            // 刷新菜单
        }
    };

    var macroCommand = MacroCommand();
    macroCommand.add(loadImgCommand);
    macroCommand.add(refreshMenuCommand);

    var setCommand = function(id, command) {
        document.getElementById(id).onclick = function() {
            command.exec();
        };
    };
    window.onload = function() {
        setCommand("both", macroCommand);
    };
</script>

这样，点击按钮之后，就会遍历根节点下的所有叶子节点并且执行他们各自的exec()方法了，因此暴露相同的exec()方法很重要。同时，为了确保组合节点和叶子节点之间差异的透明性，我们需要为叶子节点也添加add()方法，只是调用的时候会抛出错误，可以帮助用户在错误调用该方法的时候尽快找到问题所在。
命令模式和组合模式组成的宏命令系统的结构可以更加复杂，节点可以无限延伸下去，组合节点可以包含任意多的组合节点和叶子节点，因此就像一棵树一样。另外在上面例子的基础上还可以扩展出很多的用法，例如给组合节点新增删除节点的方法、让子节点可以引用父节点等等。

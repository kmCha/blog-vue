javascript设计模式之单例模式(singleton)

发表于 2016-01-08   |   分类于 javascript   |  

单例？

所谓单例就是单一实例的意思。在传统面向对象语言中，对象要由类来创建，而单例模式意思就是用那个类只能创建出一个实例，多了没有（多次创建返回的是第一次创建的那个实例）。由于javascript没有类，因此问题又简化了一些，我们可以直接编写函数返回对象来实现单例模式。

实现

知道了单例是个神马意思之后要实现他思路也就清晰了，无非就是定义一个函数，用闭包保存对象创建的状态，如果没有创建过则创建对象，并且保存。如果创建过就直接返回保存的对象。

普通单例模式

拿创建一个div元素举例，如果我们要创建一个页面中唯一的div，比如说网页的登录框，那么可以这样子写：

var createLogin = function() {
    var div = null;
    return function(html) {
        if(!div) {
            div = document.createElement("div");
            div.innerHTML = html;
            div.style.display = "none";
            document.body.appendChild(div);
        }
        return div;
    }
}();
loginButton.onclick = function() {
    var login = createLogin("我是登录框");
    login.style.display = "block";
};

上面的代码可以实现单例模式，每次点击loginButton（假设存在）的时候就只会返回createLogin()函数第一次创建的保存在闭包中的那个div而不会重复创建。但是既然这是第一个例子那么注定是个炮灰，他有啥缺点呢？

我们看到，createLogin()函数中既处理了创建div元素的事情，又负责了判断div到底存不存在的逻辑，不符合“单一指责原则”。那有些人要说了，别给我扯些啥原则不原则的，你还怕函数累着了怎么的，没有实际的影响吗？
不好意思还真有，你想，这里是创建一个唯一的div，那么如果你要继续创建一个唯一的iframe来显示第三方页面，你怎么做？是不是要复制粘贴那段判断的逻辑代码然后放在createIframe()函数中？
这就是问题所在，设计模式的核心就是要将会改变的和不会改变的代码分离开来，进而增强代码的复用性。因此，正确的打开方式应该是下面的例子。

文艺单例模式

同样如果我们要创建一个登录框：

var singleton = function(fn) {
    var instance = null;
    return function() {
        return instance || instance = fn.apply(this, arguments);
    }
};
var createLogin = function(html) {
    var div = document.createElement("div");
    div.innerHTML = html;
    div.style.display = "none";
    document.body.appendChild(div);
    return div;
};

var createSingleLogin = singleton(createLogin);

loginButton.onclick = function() {
    var login = createSingleLogin("我是登录框");
    login.style.display = "block";
};

这样，创建元素和判断是否创建过两件事就分离开来了，现在如果我们要想创建一个唯一的iframe元素，就可以像这样：

var createIframe = function(src) {
    var frame = document.createElement("iframe");
    frame.src = src;
    frame.style.display = "none";
    document.body.appendChild(frame);
    return frame;
};

var createSingleIframe = singleton(createIframe);

frameBotton.onclick = function() {
    var frame = createSingleIframe("www.kmhaoshuai.com");
    frame.style.display = "block";
}

现在想创建任何单一对象都可以用这个方法了。

单例模式不只能用于创建单例

在上面实现单例模式的代码中，singleton()函数是关键，他接收一个函数当作参数。在上面的例子中，这个函数是用来创建一个对象并且返回的。返回的对象保存到了闭包中的instance变量中。那么问题来了，what if 传进singleton()函数的函数不是用来创建对象用的？
其实这个问题很好回答，看singleton()函数的实现原理就能很容易的发现他是根据instance变量中的值来判断对象是否被创建过了，或者，说得更准确一点，他是根据instance变量中的值来判断传入的函数是否被调用过了。如果没被调用，则让instance的值等于函数的返回值，在上面的例子中，为一个对象。那么要是函数不是返回对象用的，我们只用让他在最后返回一个真值，例如true，就能同样使用singleton()来让函数只能执行一次。拿绑定事件处理函数的操作来说，singleton()实现的效果相当于用jQuery的$().one()函数的效果：

var bindEvent1 = function() {
    $('#div').one('click', function(){
        alert('click');
    });
};

var bindEvent2 = singleton(function() {
    document.getElementById("div").onclick = function() {
        alert('click');
    };
    return true;
});

上面两个函数的效果是相同的。

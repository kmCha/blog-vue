邪恶两兄弟：eval()和with

发表于 2016-01-04   |   分类于 javascript   |  

这两兄弟是啥

eval()函数

eval()是一个能够接收一个字符串作为参数的函数，能够在他所在的词法作用域中动态执行代码。比如说在程序的某个地方，你觉得之后会在运行时根据那时候的状态来执行某些语句，而不是在代码创建的时候就能确定要执行的代码，那么eval()就笑了：“朋友不用我还在等什么”

with关键字

with是个javascript中的关键字，主要解决某些地方需要重复引用对象的问题。比如说你需要执行类似下面的代码：

var lalala = {
    a: 1,
    b: 2,
    c: 3
};
lalala.a = 4;
lalala.b = 5;
lalala.c = 6;

那么要是你用with的话就可以写成：

var lalala = {
    a: 1,
    b: 2,
    c: 3
};
with(lalala) {
    a = 4;
    b = 5;
    c = 6;
}

嗯，看上去也相当美味可口。但是事实上不管是网络上还是各种书籍里，甚至是javascript标准委员会都不待见他们。

好好的怎么就被通缉了

看上去不错的两个功能怎么这么不受待见？不管是各类文章还是各种书籍，只要是关于javascript的，就叫你不要用这俩功能。在ES5中加入的严格模式对eval()进行了限制，对with直接就禁止使用了。下面来看看这些都是为什么。

eval()犯什么法了

eval()虽然能运行时求值，但是仔细想想能用到运行时求值的时候多吗，怕是不多吧，更何况他还能轻易改变他所在的作用域：

function foo(text) {
    eval(text);
    console.log(a);
}
var a = 1;
foo("var b = 1;");     // 1
foo("var a = 2;");  // 2

可以看到，eval()执行了var a = 2;之后，相当于在代码创建的时候var a = 2;就在那里，因此console.log(a)会输出2。因此，想在运行时修改程序中的任何词法作用域就变得非常简单，这是非常危险的。这是eval()函数邪恶的地方之一，还有一个重要的方面是对性能的影响，放在后面和with一起介绍。

with又招谁惹谁了

招谁惹谁没事，关键是with惹到javascript运行环境的哥老官全局对象了。直接看例子：

function go(obj) {
    with(obj) {
        a = 2;
    }
}
var foo = {
    a: 1
},
    bar = {
    b: 1
};

go(foo);
console.log(foo.a); // 1
console.log(a);     // undefined

go(bar);
console.log(bar.a); // undefined
console.log(a);        // 3(对头，a变成了一枚金闪闪的全局变量)

不难看出，当用with关键字时，当传入的对象存在跟下面要赋值的标识符同名的属性时，with会按我们预期完成任务。但是如果不存在咋办呢，不存在的时候with居然去碰了全局对象，你说他该不该被一枪崩了？就光是这一个理由就够他吃枪子了，但是事情还没完，他也会和eval()一样影响javascript的运行性能。

对性能的影响

要分析他们对性能的影响，就要从较为底层的编译器说起。javascript是一门编译型语言，但是他的特殊性，他跟其他的编译型语言在编译过程中又有一些不同。我们写好的javascript代码会在执行之前被编译器编译（I mean, exactly之前，一般情况下编译后就立即执行），在编译过程（词法分析、语法分析、代码生成）中的词法分析阶段，编译器会对词法作用域中的标识符的访问方式做优化，让我们的代码在运行时能够更快的访问到需要访问的标识符（变量、函数等）。编译器能这样做的前提是在代码编写之后各个标识符都是已经确定了的，这样他就能定位我们的每个标识符然后通过某些手段来优化访问他们的过程。但是正如我们上面讨论过的，eval()和with会在代码运行时改变我们的词法作用域，因此编译器是没办法提前知道他们哥俩要加入的标识符的名字和位置是什么的，所以也就没办法在代码编译的过程中对他们进行优化。更严重的是因为他们能够改变程序的作用域，所以编译器就算之前对作用域做了优化，在作用域改变之后，那些优化都会付诸流水，所以在大量使用eval()和with之后，程序运行会变的特别的慢。
虽然都知道要远离他们，但是知道个所以然还是多少会对理解javascript这门语言有帮助的。
